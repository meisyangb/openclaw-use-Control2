# Conversation Memory Manager

专门用于管理长对话上下文、保持对话连贯性的技能，防止因上下文过长导致的性能问题。

## 核心功能

### 1. 智能上下文管理
- **动态压缩**: 自动识别和压缩不重要的对话历史
- **关键信息保留**: 保留对后续对话有价值的核心信息
- **会话分段**: 将长对话按主题或时间分段管理
- **重要性评估**: 基于相关性、新鲜度和重要性筛选内容

### 2. 记忆存储优化
- **分层存储**: 
  - 短期记忆：当前对话的上下文
  - 中期记忆：项目相关的关键信息
  - 长期记忆：用户偏好、工作习惯等持久化信息
- **智能清理**: 定期清理过时或不相关的记忆
- **记忆索引**: 建立关键词索引，快速检索相关信息

### 3. 上下文窗口控制
- **窗口管理**: 根据模型限制动态调整上下文窗口大小
- **优先级排序**: 确保最重要的信息保持在上下文中
- **渐进式遗忘**: 按照时间衰减和重要性逐步移除旧信息
- **关键锚点**: 保留对话中的关键决策点和重要成果

### 4. 对话连贯性维护
- **主题跟踪**: 追踪当前对话主题和子主题
- **引用解析**: 正确理解和使用之前对话中的引用
- **一致性检查**: 确保回复与之前对话保持逻辑一致
- **上下文提示**: 在需要时主动请求澄清或提供背景信息

## 适用场景

### 长项目讨论
- 跨多个会话的复杂项目讨论
- 持续迭代的代码审查和优化
- 长期的技术决策和架构规划

### 重复性任务
- 连续的bug修复和功能开发
- 持续的学习和技能提升
- 定期的项目状态更新

### 多轮对话
- 深入的技术问题讨论
- 架构设计和决策过程
- 代码重构和优化建议

## 工作流程

1. **对话开始**: 初始化新会话的记忆系统
2. **内容分类**: 实时分析并分类对话内容
3. **权重评估**: 为不同内容分配重要性权重
4. **存储优化**: 根据权重和时效性更新记忆
5. **上下文构建**: 根据当前需求构建合适的上下文窗口
6. **一致性检查**: 确保回复与记忆保持一致

## 技术实现

### 内存层级
```python
# 概念结构
Memory = {
    "short_term": [],      # 当前对话上下文 (限制大小)
    "medium_term": [],    # 项目相关关键信息 (主题化存储)
    "long_term": {},      # 用户偏好和习惯 (结构化存储)
    "context_window": []   # 实际传递给模型的上下文
}
```

### 重要性算法
- **新鲜度**: 时间衰减因子
- **相关性**: 与当前任务的相关程度
- **独特性**: 信息在其他地方是否可获取
- **行动价值**: 对后续决策和行动的影响程度

### 压缩策略
- **语义压缩**: 保留核心语义，去除冗余
- **总结提取**: 自动生成对话摘要
- **关键词提取**: 识别并保留关键术语和概念
- **结构化存储**: 按主题和类别组织信息

## 性能优化

### 计算效率
- **增量处理**: 只处理新增的对话内容
- **异步更新**: 后台更新记忆系统，不影响实时对话
- **缓存策略**: 缓存常用查询和计算结果
- **批量操作**: 集中处理多个记忆更新

### 内存优化
- **智能采样**: 在保持质量的前提下减少存储
- **过期清理**: 定期清理过期和无用信息
- **压缩算法**: 使用高效的压缩算法存储历史
- **分页加载**: 按需加载相关记忆内容

## 限制与边界

### 不适用场景
- 简单的单轮问答
- 完全独立的技术文档阅读
- 超大规模的实时数据处理

### 资源限制
- 内存使用受系统限制
- 处理复杂度影响实时性能
- 长期记忆维护需要平衡存储空间
