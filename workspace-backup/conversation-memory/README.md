# Conversation Memory System - Usage Guide

## 概述

对话记忆管理系统是一个智能的上下文管理技能，专门设计用于处理长对话中的上下文保持问题，防止因上下文过长导致的性能下降。

## 核心特性

### 1. 智能记忆分层
- **短期记忆**：当前对话的上下文，实时处理
- **中期记忆**：项目相关的关键信息，按主题存储
- **长期记忆**：用户偏好和工作习惯，持久化存储

### 2. 动态上下文管理
- **自动压缩**：识别并压缩不重要信息
- **重要性评分**：基于相关性、新鲜度、独特性和行动价值
- **窗口优化**：根据重要性动态调整上下文窗口大小

### 3. 对话连贯性维护
- **主题跟踪**：追踪对话主题变化
- **引用解析**：正确理解和使用之前的引用
- **一致性检查**：确保回复与之前对话保持逻辑一致

## 工作原理

### 重要性评分算法
基于四个主要因素：
- **相关性**：与当前任务的相关程度 (40%)
- **新鲜度**：时间衰减因子 (20%)
- **独特性**：信息在其他地方是否可获取 (20%)
- **行动价值**：对后续决策和行动的影响程度 (20%)

### 压缩策略
- **高重要性信息**：100%保留
- **中等重要性信息**：50-80%压缩，保留核心语义
- **低重要性信息**：80-90%压缩，仅保留关键点
- **重复信息**：去重，保留最新版本

### 上下文构建
1. 当前对话内容（100%保留）
2. 中期记忆中与当前主题相关的内容（优先保留）
3. 长期记忆中用户偏好的内容（选择性保留）
4. 历史决策和关键信息（保留关键点）

## 性能优化特性

### 计算效率
- **增量处理**：只处理新增内容，不重复处理历史
- **异步更新**：后台更新记忆系统，不影响实时对话
- **缓存策略**：缓存常用查询和计算结果

### 内存管理
- **智能采样**：在保持质量的前提下减少存储
- **过期清理**：定期清理过期和无用信息
- **分页加载**：按需加载相关记忆内容

## 配置参数

### 内存配置
- **短期记忆容量**：5,000 tokens
- **中期记忆容量**：10,000 tokens
- **长期记忆容量**：20,000 tokens

### 压缩配置
- **高重要性保留比例**：100%
- **中重要性保留比例**：70%
- **低重要性保留比例**：30%

### 刷新配置
- **刷新间隔**：每5个消息或上下文使用率超过80%
- **处理超时**：1秒内完成记忆更新
- **内存使用限制**：不超过系统可用内存的80%
