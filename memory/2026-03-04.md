## 💰 Cost Optimization Strategy (Critical!)

### Billing Models Understanding

**阿里云百炼 (bailian)**
- **计费方式**: 按请求次数计算
- **优化策略**: 适合复杂任务，单次请求可以处理更多内容
- **最佳使用**: 需要深度分析、长文本处理的任务

**智谱 AI (zai)**
- **计费方式**: 按 token 用量计算
- **优化策略**: 需要精确控制 token 消耗，避免浪费
- **风险**: token 用完就停止服务
- **最佳使用**: 简单对话、短文本任务

### Model Selection Rules

1. **简单对话/短回复** → 优先使用 zai/glm-4.5-air (轻量、省 token)
2. **复杂分析/长文本** → 优先使用 bailian/qwen3.5-plus (按次计费，不限制 token)
3. **图像理解** → bailian/qwen3.5-plus 或 zai/glm-4.6v (根据任务复杂度)
4. **代码任务** → bailian/qwen3-coder-next 或 qwen3-coder-plus
5. **日常对话** → 尽量简洁，减少 token 消耗

### Token Conservation Tactics

- 回复简洁，避免冗余
- 只在必要时调用大模型
- 简单任务使用轻量模型
- 避免重复调用相同内容
